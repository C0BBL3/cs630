# An Analysis and Gamification of Networking in Data Centers

## 1. Introduction

The way computers are wired together inside a datacenter has a huge impact on how well they perform. A bad network layout can leave expensive hardware sitting idle while data waits to move between machines. In distributed AI training, every node needs to share its results with every other node, and if the network becomes the bottleneck, 30-50% of training time can be lost just to communication.

Despite how much this matters, students rarely get hands-on experience with network design. They see diagrams in a textbook and memorize which topologies are considered good, but they never get to build one and see how it actually performs.

Consider a simple example: if you wire 25 computers in a ring, every piece of data has to hop through all 25 nodes one at a time, creating a massive bottleneck. Wire them as a tree instead and data flows in parallel, finishing much faster. That difference is easy to understand once you see it happen, but nearly impossible to appreciate from a static diagram.

**Research question:** Can a gamified, interactive network topology designer, where users build datacenter interconnects under realistic constraints and receive immediate performance feedback, teach the relationship between network structure and distributed computing performance?

## 2. State of the Art

Existing work in this space falls into three categories that don't really overlap.

Researchers have proposed several datacenter network designs. Al-Fares et al. [1] introduced the fat-tree, which uses cheap off-the-shelf switches to maximize throughput. Kim et al. [2] proposed the dragonfly, which cuts down the number of hops data has to travel. Singla et al. [3] showed that even random wiring patterns can beat carefully engineered ones. Singh et al. [4] documented how Google built and evolved its datacenter networks over a decade. These papers are foundational, but they're written for experts and don't try to build intuition.

On the simulation side, tools like ns-3 [5], FatTreeSim [6], and Wormhole [7] let researchers test network designs by simulating traffic in detail. They're effective, but they're research-grade tools that require significant expertise and aren't designed for learning.

There are also educational tools like Conquer the Net [8] and NetEmulator [9] that teach networking through interactive exercises, but they focus on configuring individual devices like setting IP addresses and routing tables. Neither one lets you design the network layout itself or shows how topology affects overall performance.

That leaves a clear gap: there is no tool that lets you design a network, run traffic through it, and see how your design choices affect performance.

## 3. Proposed Solution

The idea behind this project is that you learn network design by building networks and watching them succeed or fail. When you connect nodes in a ring and see packets pile up at a bottleneck while your score drops, you start to understand why that layout doesn't work. When you add one extra link and your score jumps, you see firsthand why it helps. That kind of feedback is hard to get from a lecture.

We are building a browser-based game with three main components.

The first is an interactive editor where you place nodes and draw connections between them. Early levels lock nodes to a 2D grid so you can focus on wiring. Later levels let you place nodes freely, and the final levels move into full 3D.

The second is a simulation engine that models real network traffic. Every link has bandwidth and delay, and packets queue up at congested links just like they would in a real network. The engine tries multiple communication strategies and scores your design based on the best one. Scores are deterministic and consistent across every browser and platform.

The third is a progressive level system. Eight levels ramp from 4 unconstrained nodes up to 25 nodes under every constraint at once: connection limits, cable length, total cable budget, and planarity. Each completed level unlocks an encyclopedia entry about a real datacenter topology, showing how your design compares.

The entire application runs in a browser with no installation and no backend. You open a single HTML file and start playing.

### Milestones

1. Core graph model: node/edge data structures, shortest path, diameter, bisection bandwidth.
2. Simulation engine: packet-level discrete-event simulation, ring and tree all-reduce, scoring formula.
3. 2D editor and levels 1-3: fixed-position editor, constraint validation, first playable levels.
4. Scoring visualization: animated packet flow, bottleneck highlighting, score breakdown, star rating.
5. Advanced levels 4-6: free placement, cable budget, planarity constraints.
6. 3D mode and levels 7-8: Three.js integration, 3D editor.
7. Encyclopedia and polish: unlockable topology entries, tutorial overlays, UI polish.
8. Academic paper: system design analysis, topology analysis, results.
